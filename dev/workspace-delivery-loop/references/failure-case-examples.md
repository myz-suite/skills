# Failure Case Examples

## 示例 1：必填缺失

**前置条件**：用户未填写必填字段  
**触发**：点击提交  
**预期**：
- 显示错误提示 “字段 X 不能为空”
- 不发起请求
- 保持原输入内容

## 示例 2：权限不足

**前置条件**：当前用户无权限  
**触发**：访问受限页面或执行操作  
**预期**：
- 提示 “无权限”
- 不显示敏感数据
- 记录可追踪的日志/事件

## 示例 3：网络失败

**前置条件**：网络断开或请求超时  
**触发**：执行保存/同步操作  
**预期**：
- 显示失败提示
- 提供重试入口
- 保留本地未提交状态

## 示例 4：后端返回错误码

**前置条件**：服务返回 4xx/5xx  
**触发**：提交请求  
**预期**：
- 统一错误提示（含可读文案）
- 不更新本地成功状态
- 记录错误信息供定位
