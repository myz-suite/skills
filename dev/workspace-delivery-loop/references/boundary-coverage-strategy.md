# Boundary Coverage Strategy

## 覆盖优先级

1. **数据边界**：空值、最小/最大、超长、非法格式  
2. **权限边界**：未登录、无权限、过期会话  
3. **系统边界**：网络失败、超时、重试、重复提交  
4. **并发边界**：多次点击、并行请求、重复写入  

## 覆盖方法

- 每个核心用例至少覆盖 2 个边界条件  
- 对写操作必须包含“失败路径”  
- 对读操作必须包含“无数据/空状态”  

## 记录方式

在验收标准或用例中显式标记边界：
- `Boundary:` 空值  
- `Boundary:` 最大长度  
- `Boundary:` 权限不足  

## 最小覆盖示例

用例：创建记录  
- 主路径：创建成功  
- 边界 1：标题为空 → 禁止提交  
- 边界 2：标题超长 → 提示并阻止  
- 失败路径：网络超时 → 提示重试
