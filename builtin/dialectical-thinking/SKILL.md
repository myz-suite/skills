## 适用场景

- 两种或多种观点冲突且都具备合理性
- 需要在价值、伦理、成本、体验之间做深度权衡
- 目标是“综合”而非“站队”

## 核心目标

用“正题-反题-合题”的辩证结构，找出高阶解法或更具包容性的结论。

## 思维模型（Dialectical Model）

1. **正题（Thesis）**：提出主流或直觉上的核心观点。
2. **反题（Antithesis）**：提出有力的反向观点或反例。
3. **矛盾分析**：识别对立观点之间的张力与冲突根因。
4. **合题（Synthesis）**：寻找更高层次的综合方案。
5. **验证与边界**：明确综合方案适用条件与代价。

## 思维步骤（一步一步执行）

1. **陈述正题**  
   - 输出：正题结论 + 支撑理由。
2. **提出反题**  
   - 输出：反题结论 + 支撑理由。
3. **定位矛盾核心**  
   - 输出：冲突根因（资源/价值/结构/信息）。
4. **提出综合解法**  
   - 输出：高阶方案或折中策略。
5. **验证与边界条件**  
   - 输出：适用条件、代价与风险。

## 引导模式（提问与提示）

### 正题与反题
- “最强支持与最强反对的观点分别是什么？”
- “双方背后的价值诉求是什么？”

### 矛盾分析
- “冲突的根因是资源稀缺还是价值冲突？”
- “是否存在结构性限制导致两者无法兼得？”

### 综合与验证
- “有没有更高层次的目标能同时容纳两方？”
- “综合方案的代价是什么？”

## 输出结构模板（建议格式）

```
正题:
反题:
矛盾核心:
合题/综合方案:
边界与代价:
```

## 约束规则

- 必须呈现正题与反题，不可只输出单方观点。
- 综合方案需说明代价与适用条件。
- 若无法综合，明确“冲突不可调和”的原因。
