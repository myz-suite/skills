## 适用场景

- 需要明确因果关系、变量影响、权衡成本收益的任务
- 业务/技术方案对比、策略评估、指标诊断、风险分析
- 用户希望得到结构化结论或可执行的决策建议

## 核心目标

以“问题 → 变量 → 证据 → 结论”的分析链条，输出**可追溯、可复核、可执行**的判断。

## 思维模型（Analytical Model）

1. **问题定义**：目标是什么？成功衡量是什么？
2. **变量拆解**：关键变量/驱动因素/约束条件有哪些？
3. **关系建模**：变量之间的因果、相关、依赖关系是什么？
4. **证据核验**：数据、经验、假设的可信度如何？
5. **方案对比**：在相同约束下的方案优劣与权衡。
6. **结论输出**：给出明确结论与下一步行动。

## 思维步骤（一步一步执行）

1. **明确问题边界**  
   - 将“模糊问题”改写成“可验证问题”。  
   - 输出：目标、范围、约束、成功指标。
2. **拆解变量与假设**  
   - 列出影响结果的主要因素和隐含假设。  
   - 输出：变量清单、关键假设清单。
3. **建立因果链或逻辑框架**  
   - 用“如果/那么”或“驱动→结果”的逻辑表达变量关系。  
   - 输出：简洁的关系图（文字版）。
4. **收集与核验证据**  
   - 标注证据的来源、时效、可信度。  
   - 输出：证据清单 + 可信度评级。
5. **方案/结论评估**  
   - 以指标/成本/风险/收益维度进行比较。  
   - 输出：对比表或排序理由。
6. **行动建议与验证计划**  
   - 给出可执行下一步与验证方式。  
   - 输出：行动步骤 + 复核指标。

## 引导模式（提问与提示）

### 关键澄清
- “你的最终目标是什么？如何判定成功？”
- “有哪些明确约束（时间/成本/资源/合规）？”

### 变量拆解
- “这个结果由哪些关键因素驱动？”
- “最可能改变结论的变量是什么？”

### 证据核验
- “这条结论基于哪些数据或经验？”
- “证据来源是否具有时效性与代表性？”

### 权衡与选择
- “方案之间的成本、收益、风险各是什么？”
- “在当前约束下，最可行的选择是哪一个？”

## 输出结构模板（建议格式）

```
问题定义:
变量与假设:
关系与因果链:
证据与可信度:
方案对比:
结论与行动:
```

## 约束规则

- 不凭空下结论，必须说明依据或不确定性。
- 不跳过“变量/证据/权衡”三步。
- 若证据不足，明确标注并提出补充信息需求。
