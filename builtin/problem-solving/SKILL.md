## 适用场景

- 问题复杂但缺乏清晰路径
- 需要系统化分析并快速收敛到可执行方案
- 面对反复出现的问题，需要找到根因

## 核心目标

形成“问题 → 根因 → 方案 → 执行 → 复盘”的闭环。

## 思维模型（Problem Solving Model）

1. **问题定义**：明确问题与成功标准。
2. **根因分析**：找出导致问题的核心原因。
3. **方案生成**：提出多个可选方案。
4. **决策与执行**：选择最优方案并落实。
5. **复盘改进**：验证效果并持续优化。

## 思维步骤（一步一步执行）

1. **问题界定**  
   - 输出：问题描述 + 影响范围 + 目标。
2. **根因分析**  
   - 使用 5 Whys / 鱼骨图 / 对比分析。  
   - 输出：根因列表与证据。
3. **方案生成**  
   - 输出：不少于 3 个备选方案。
4. **方案评估与决策**  
   - 输出：成本/收益/风险对比与选择理由。
5. **执行计划**  
   - 输出：步骤、责任、时间线。
6. **效果验证与复盘**  
   - 输出：验证指标与复盘动作。

## 引导模式（提问与提示）

### 定义问题
- “我们要解决的核心问题是什么？”

### 根因分析
- “为什么会发生？再往下一层原因是什么？”
- “是否有数据支持这个根因？”

### 方案与执行
- “最小可行方案是什么？”
- “实施成功的指标是什么？”

## 输出结构模板（建议格式）

```
问题定义:
根因分析:
方案清单:
方案评估与决策:
执行计划:
验证与复盘:
```

## 约束规则

- 根因必须可验证，避免主观推断。
- 方案不少于 3 个。
- 必须包含验证与复盘机制。
