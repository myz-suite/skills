## 适用场景

- 需要从目标落地为执行计划
- 项目启动、路线图制定、跨团队协作
- 需要明确里程碑与监控机制

## 核心目标

输出一份可执行、可监控、可调整的计划，涵盖时间、资源与风险。

## 思维模型（Planning Model）

1. **目标与成功标准**：明确要达到的结果。
2. **里程碑拆解**：把目标拆分为阶段性成果。
3. **资源与依赖**：人力、预算、工具、外部依赖。
4. **风险与缓冲**：识别风险并设置应对策略。
5. **监控与迭代**：指标与复盘机制。

## 思维步骤（一步一步执行）

1. **明确目标与边界**  
   - 输出：目标 + 成功标准。
2. **拆解里程碑**  
   - 输出：里程碑列表 + 时间节点。
3. **任务与资源分配**  
   - 输出：任务清单 + 资源需求。
4. **风险与应对**  
   - 输出：风险清单 + 应对措施。
5. **监控指标**  
   - 输出：进度/质量/成本指标。
6. **迭代机制**  
   - 输出：复盘节奏 + 调整规则。

## 引导模式（提问与提示）

### 目标与范围
- “最终交付物是什么？成功标准是什么？”

### 里程碑
- “哪些阶段性成果能证明我们在前进？”

### 风险与监控
- “最可能失败的环节在哪里？”
- “如何在早期发现偏差？”

## 输出结构模板（建议格式）

```
目标与成功标准:
里程碑与时间线:
任务与资源:
风险与应对:
监控指标:
迭代机制:
```

## 约束规则

- 必须包含里程碑与监控指标。
- 计划应可执行，避免抽象化表述。
- 若资源不足，明确优先级与削减方案。
