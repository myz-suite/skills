## 适用场景

- 多阶段任务规划、复杂问题求解、需要逐步推进的决策
- 存在不确定性，需要探索多个路径
- 任务可能随新信息发生修订

## 核心目标

把思考拆分为可执行步骤，记录分支与修订，让决策过程透明且可迭代。

## 思维模型（Sequential Model）

1. **主线步骤**：按照“目标 → 分解 → 执行 → 校验”推进。
2. **分支探索**：对关键决策点同时探索多个路径。
3. **修订机制**：根据新信息回滚或改写先前步骤。

## 思维步骤（一步一步执行）

1. **定义目标与完成条件**  
   - 输出：明确目标 + 结束条件。
2. **拆解任务路径**  
   - 输出：主线步骤清单（Step 1..n）。
3. **识别关键决策点**  
   - 输出：可能需要分支的节点。
4. **分支探索（Branching）**  
   - 输出：每个分支的假设、收益、风险。
5. **选择路径并执行**  
   - 输出：选中分支 + 执行动作。
6. **检查与修订（Revising）**  
   - 输出：修订原因 + 被修订的步骤 + 新结论。

## 分支与修订的记录格式

### 分支记录
```
分支点: Step X
分支A: 假设 / 优势 / 风险 / 触发条件
分支B: 假设 / 优势 / 风险 / 触发条件
```

### 修订记录
```
修订触发: 新信息/验证失败/约束变化
影响步骤: Step X, Step Y
修订内容: 旧结论 -> 新结论
```

## 引导模式（提问与提示）

### 步骤拆解
- “我们完成这个目标需要哪几步？”
- “哪一步最不确定？需要先验证什么？”

### 分支探索
- “如果从不同假设出发，会有哪几条路径？”
- “每条路径的最大风险是什么？”

### 修订触发
- “有什么新信息会迫使我们回滚？”
- “当前结论是否需要更新？”

## 输出结构模板（建议格式）

```
目标与完成条件:
主线步骤:
分支探索:
路径选择:
执行与检查:
修订记录:
```

## 约束规则

- 每次仅推进一个步骤，保持可追踪。
- 必须显式标记分支与修订原因。
- 若信息不足，优先生成可验证的下一步。
